# Capability Matrix — AI QA Factory v5.0.8 Model Routing Profiles

AI QA Factory is a broad opportunity router with focused execution.

## Strong execution

| Area | Support | Outputs |
|---|---|---|
| Playwright + TypeScript | Strong | scaffold, tests, CI notes, review |
| Web QA / SaaS QA audit | Strong | QA plan, risk audit, bug report pack |
| AI-native exploratory QA | Strong | proposal, screening answers, evidence checklist, QA plan |
| SaaS multi-tenant / billing / auth | Strong | responsible black-box audit plan, commercial strategy |
| API testing | Good | API smoke notes/tests |
| Flaky regression automation | Strong | root-cause review, failing-test strategy |
| Selenium → Playwright migration | Strong/advisory | migration plan, starter scaffold |

## Supported / adjacent

| Area | Support | Guardrail |
|---|---|---|
| React Native / Maestro release QA | Conditional | confirm Mac/devices/TestFlight/Emulators |
| Next.js / React app QA | Supported | QA/UX/automation angle, not full dev unless verified |
| PWA/offline testing | Supported | clarify devices and test data |
| UX walkthrough / recorded testing | Supported | product feedback, not automation-first |
| Technical writing / documentation | Adjacent strong | provide real writing sample; check AI policy |
| AI workflow automation | Adjacent | separate from core QA; discovery-first |

## Advisory only

| Area | Support | Guardrail |
|---|---|---|
| Tosca / Tricentis | Advisory only | do not claim expert Tosca hands-on unless verified |
| Formal security / pentest | Advisory only | responsible discovery only, no exploit claims |
| Deep performance/load testing | Advisory/basic | clarify load model/environment |
| Compliance testing | Advisory | no regulated-compliance claims without scope |
| POS hardware | Advisory | requires client-provided hardware/access |

## Skip / risk

- Low-value $5 / $10 tests that damage Senior QA positioning.
- Crypto/deposit/ID testing.
- Pure full-stack developer roles not aligned with QA/SDET.
- Platform-policy risks or AI evaluator tasks where AI use is banned for the task itself.

## Core rule

Broad intelligence. Focused execution. Honest routing.

Self-health is intentionally safe and local: it can detect inconsistencies and suggest repair steps, but it must not auto-submit, auto-push, run destructive tests, make payments, or change client systems.
